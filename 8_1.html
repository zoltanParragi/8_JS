Variable hoisting, operators, branches, loops

variable lift/hoisting

In JS function and the variable declaration will be raised to the top of the function. Variables can be declared after use OR before the first use.

Function declarations and variable declarations will always be the interpreter quietly been "promoted" to the top of the method body

But JavaScript variable declarations will only increase, not initialized.

branches (if, switch)

Example 1 - no overlap in cases, independent cases

        let h = 2;
        switch(h) {
            case 1:
                console.log(1);
            break;
            case 2:
                console.log(2);
            break;
            case 3:
                console.log(3);
            break;
            default:
                console.log("something");
            break;
        }

the same with if

        if(h===1) { //the same as previous switch
            console.log(1+" with if");
        } else if (h===2) {
            console.log(2+" with if");
        } else if (h===3) {
            console.log(3+" with if");
        } else {
            console.log("something"+" with if");
        }

Example 2, not independent cases, overlap

        let i = 8;
        switch(true) {
            case i<5:
                console.log("smaller than 5, switch");
            case i<9:
                console.log("smaller than 9, switch");
            case i<15:
                console.log("smaller than 15, switch");
            default:
                console.log("smaller than infinite, switch");
        }

the same with if:

        if(i<5) {
            console.log("smaller than 5, if");
        }
        if(i<9) {
            console.log("smaller than 9, if");
        }
        if(i<15) {
            console.log("smaller than 15, if");
        }
        if(i<Infinity) {
            console.log("smaller than infinity, if");
        }
